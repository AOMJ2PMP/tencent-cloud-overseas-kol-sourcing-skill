# Output Contract

## 决策摘要

- 推荐策略：头部 + 中腰部 / 多位垂类 / 单一权威等
- 预算结论：可用达人预算和预留比例
- 首选组合：人选、scope、预计费用、选择理由
- 最大不确定性：报价、地域、档期、产品权限等

## Product Truth Card

使用 product-research.md 的字段。关键产品 claim 必须带官方来源。

## 主推候选表

至少包含：

| 字段 | 说明 |
| --- | --- |
| Creator / URL | 平台直达链接 |
| Base / audience geo | 分开写，附证据 |
| Platform / language | 近期主要内容 |
| Followers | TikHub 抓取值和日期 |
| Median performance | 样本量、格式、时间窗 |
| Relevant examples | 至少 2-3 条链接 |
| Sponsored evidence | 合作品牌/内容链接/日期 |
| Product angle | 为该达人定制的自然选题 |
| Collaboration likelihood | 1-5 + 理由 |
| Quote low/base/high | 原币和预算币种 |
| Score / confidence | 总分和证据置信度 |
| Risks / next verification | 待询价、受众、排他、档期等 |

## 推荐组合

每个组合说明：

- 人选与各自角色
- 建议 deliverables
- low/base/high 合计
- 在总预算中的剩余缓冲
- 为什么比单纯找大号更适合
- 谈判底线和可缩 scope

## 淘汰与 Wildcard

列出 3-10 个容易被反复提起的知名账号，并明确：

- 不符合 strict base
- 预算明显不匹配
- 内容与产品不自然
- 合作概率低
- 受众不匹配
- 证据不足

## 输出语言

- 内部判断、风险、预算和备注：中文。
- 给达人看的选题、brief 和 outreach：英语或目标语言。
- 对不确定信息使用“预计、推测、待确认”，不写成已确认事实。
