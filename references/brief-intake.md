# Brief Intake

## 目标

把模糊的“帮我找达人”变成可执行的筛选约束。问题要少而关键，每轮 2-3 个，必要时继续第二轮。

## 必须确认

| 字段 | 为什么会改变名单 | 示例 |
| --- | --- | --- |
| 产品与链接 | 决定用户、内容难度和 claim | 产品页、文档、GitHub |
| Campaign 目标 | 声量、试用、注册、lead 的达人完全不同 | Awareness / adoption / leads |
| 目标市场规则 | 当地 base 与当地受众不是一回事 | strict Singapore-based |
| 平台与语言 | 决定数据、格式和报价 | YouTube + X, English |
| 交付 | dedicated 与 integration 价格差异很大 | 2 个教程视频 |
| 总预算与包含项 | 决定能否找头部以及组合数量 | RMB 65K 含税，不含投流 |
| 时间 | 决定档期和加急费 | 7 月底上线 |
| 内容/达人偏好 | 决定自然度 | 开发者教程、探店采访 |
| 排除项 | 避免无效研究 | 不要 agency media、不要 crypto |

## 第一轮问题

优先问以下三个：

1. 这次最重要的业务目标和可衡量结果是什么？
2. 地域要求是达人必须长期 base 当地，还是受众主要在当地即可？
3. 总预算包含哪些成本，期望几位达人和什么交付？

## 第二轮问题

视答案再问：

1. 内容语言、平台和形式是否有硬性限制？
2. 上线时间、产品体验条件、拍摄地点和审批周期是什么？
3. 有无竞品排除、品牌安全红线、使用权或投流需求？

## 可使用的安全默认值

只有用户要求直接推进时才可使用，并必须在输出中写明：

- 报价：保留 15% 执行预算。
- 授权：只含 creator 自有账号一次发布，不含 paid usage、whitelisting 和永久版权。
- 修改：1 轮脚本确认 + 1 轮事实修正。
- 地域：若未说明，不把 creator base 作为硬门槛，但单独报告受众地域证据。
- 数据：YouTube 最近 15 条同格式内容；X 最近 40 条原创内容。

## 不能默认

- 不能默认“粉丝越多越好”。
- 不能默认达人愿意为腾讯云背书。
- 不能默认竞品合作等于排他。
- 不能默认所在地等于受众所在地。
- 不能默认产品已在目标市场可购买或可访问。
- 不能默认报价包含差旅、税费、授权、投流或经纪费。
